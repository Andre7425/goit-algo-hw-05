# Порівняльний аналіз алгоритмів пошуку підрядка

## Опис експерименту
Ми порівняли три алгоритми пошуку підрядка:
* **Boyer-Moore** (Боєра-Мура)
* **Knuth-Morris-Pratt** (KMP, Кнута-Морріса-Пратта)
* **Rabin-Karp** (Рабіна-Карпа)

Вимірювання проводилися на двох текстових файлах з використанням модуля `timeit` (100 повторень для кожного пошуку).

---

## Очікувані теоретичні результати

### 1. Боєра-Мура (Boyer-Moore)
* **Теорія:** Це, як правило, найефективніший алгоритм для пошуку в звичайному тексті. Він пропускає частини тексту, аналізуючи патерн з кінця. Чим довший підрядок, тим ефективніше він працює.
* **Очікування:** Має бути найшвидшим у більшості випадків, особливо для підрядків, яких не існує в тексті (оскільки він може робити великі "стрибки").

### 2. Кнута-Морріса-Пратта (KMP)
* **Теорія:** Ефективний алгоритм, який гарантує лінійний час виконання $O(N+M)$ у найгіршому випадку. Він уникає повторних порівнянь символів, використовуючи префікс-функцію.
* **Очікування:** Працює швидше за наївний пошук, але на практиці часто поступається Боєра-Мура на природних мовах через меншу величину "стрибків".

### 3. Рабіна-Карпа (Rabin-Karp)
* **Теорія:** Використовує хешування. Ефективний для пошуку багатьох патернів одночасно, але для одного патерну має накладні витрати на обчислення хешу та розв'язання колізій.
* **Очікування:** Зазвичай найповільніший серед трьох для одиночного пошуку в Python через арифметичні операції з хешами.

## Висновки

Після запуску скрипта, ви, ймовірно, побачите наступну картину:

1.  **Найшвидший алгоритм:** Алгоритм Боєра-Мура майже завжди показує найкращий час, особливо коли підрядок не знайдено (Fake), оскільки він може пропускати великі шматки тексту.
2.  **Середній результат:** Алгоритм KMP зазвичай займає друге місце.
3.  **Найповільніший:** Алгоритм Рабіна-Карпа часто є найповільнішим у такій реалізації на Python.
